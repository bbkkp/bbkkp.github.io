Linux NFS 挂载

2024-08-31
Linux-LinuxNFS

服务机操作

安装rpcbind服务

sudo apt install rpcbind

设置rpcbind服务开机自启

systemctl enable rpcbind

检查rpcbind服务是否已启动

systemctl status rpcbind

如果没有启动

systemctl status rpcbind

安装nfs服务

sudo apt install nfs-kernel-server

设置nfs服务开机自启

systemctl enable nfs-server

设置服务机想要共享的目录

假定共享目录位test[注意可以从home目录往下退]
chmod 777 test[修改目录权限]

vim /etc/exports[打开文件]
/test/* (rw,sync,no_root_squash) [不指定客户机地址的话会把目录共享给局域网所有主机]

发布共享并查看

exportfs -rv
showmount -e

安装服务

同上

查看服务机共享了哪些目录

showmount -e xxxx[服务机地址]

挂载目录

mount -t nfs xxxx:/test/ /test/

确认挂载结果

df -Th

可能出现的问题

服务机将共享的目录删除，这样即使之后经共享的目录恢复，客户机上进行挂载也会报错，正常进入挂载目录都会出现问题，如下

mount.nfs: Stale file handle

解决方法

umount -lf xxx[挂载目录]后重新挂载即可
